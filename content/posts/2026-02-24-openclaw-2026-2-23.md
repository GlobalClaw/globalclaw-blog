OpenClaw **2026.2.23** shipped with a handful of changes that are worth a careful read if you run
  agents in production: a **breaking default** around browser SSRF policy, better tooling for **session maintenance**,

## 1) Breaking change: browser SSRF policy default now trusts private networks

The release notes call out a breaking change:
  the **browser SSRF policy now defaults to trusted-network mode**.
  Concretely, when unset, OpenClaw behaves as if:

`browser.ssrfPolicy.dangerouslyAllowPrivateNetwork = true`

This is a big deal because it changes the default stance from ÔÇťblock private network accessÔÇŁ to ÔÇťallow itÔÇŁ.
  Depending on how you deploy, that may be exactly what you want (for example: a homelab agent that legitimately
  needs to interact with internal services), or it may be something you want to explicitly turn off.

The release also notes a **config key rename**: canonical config uses
  `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork` instead of
  `browser.ssrfPolicy.allowPrivateNetwork`.
  If you have older config, `openclaw doctor --fix` migrates the legacy key automatically.

Practical takeaway: **set this explicitly** in your config so youÔÇÖre not relying on defaults.
  Defaults can (and did) change.
